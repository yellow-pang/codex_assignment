# 실시간 Car Market 향후 와이어프레임

## 1. 문서 목적

이 문서는 기존 자동차 CRUD 화면을 실시간 Car Market 서비스로 확장하기 위한 향후 화면 구조를 정리한다.

기준은 다음 문서이다.

- `docs/실시간_Car_Market_서비스_요구사항_정의서.md`
- `docs/실시간_Car_Market_향후_개발_계획서.md`
- `docs/실시간_Car_Market_UI_개선_분석_보고서.md`

현재 `docs/wireframe.md`는 기존 CRUD 앱 기준이다. 이 문서는 향후 구현할 Firebase 인증, MongoDB 차량 데이터, 차량 사진 업로드, Socket.io 상담, 관리자/상담 현황, 순수 Tailwind CSS 기반 UI를 기준으로 한다.

## 2. 전체 화면 구조

첫 화면은 차량 목록과 검색 중심으로 구성한다. 일반 사용자 화면은 카드형 차량 마켓 UI를 사용하고, 딜러/관리자 화면은 테이블형 관리 UI를 사용한다.

```text
┌────────────────────────────────────────────────────────────┐
│ Header                                                     │
│ [Car Market] [차량 검색] [상담방] [딜러 관리] [관리자]     │
│                                      [로그인/회원가입/내정보] │
├────────────────────────────────────────────────────────────┤
│ Main Content                                               │
│ - 차량 목록 / 상세 / 상담 / 딜러 관리 / 관리자 화면         │
└────────────────────────────────────────────────────────────┘
```

## 3. 사용자 역할별 화면

| 사용자 유형 | 접근 화면 | 주요 기능 |
| --- | --- | --- |
| 비로그인 사용자 | 차량 목록, 로그인, 회원가입 | 차량 검색 화면 확인, 인증 진입 |
| 일반 사용자 `buyer` | 차량 목록, 차량 상세, 상담방 목록, 상담 화면 | 차량 검색, 상세 조회, 딜러 상담 |
| 딜러 `dealer` | 차량 목록, 차량 상세, 차량 등록/수정, 상담방 목록, 상담 화면, 딜러 관리 | 차량 등록/수정, 상담 응답 |
| 관리자 `admin` | 관리자 대시보드, 차량 관리, 사용자 관리, 상담 현황 | 전체 데이터와 상담 현황 확인 |

## 4. 라우팅 구조 초안

React Router 도입을 기준으로 화면 경로를 설계한다.

| 경로 | 화면 | 접근 기준 |
| --- | --- | --- |
| `/` | 차량 목록/검색 첫 화면 | 전체 |
| `/login` | 로그인 | 비로그인 |
| `/register` | 회원가입 | 비로그인 |
| `/cars/:id` | 차량 상세 | 로그인 사용자 |
| `/dealer/cars/new` | 차량 등록 | 딜러 |
| `/dealer/cars/:id/edit` | 차량 수정 | 딜러 |
| `/chats` | 내 상담방 목록 | 로그인 사용자 |
| `/chats/:roomId` | 실시간 상담 | 로그인 사용자 |
| `/dealer` | 딜러 관리 홈 | 딜러 |
| `/admin` | 관리자 대시보드 | 관리자 |
| `/admin/cars` | 관리자 차량 관리 | 관리자 |
| `/admin/users` | 관리자 사용자 관리 | 관리자 |
| `/admin/chats` | 관리자 상담 현황 | 관리자 |

## 9. 딜러 차량 등록/수정 화면

딜러 화면은 업무형 폼으로 구성한다.

```text
┌────────────────────────────────────────────────────────────┐
│ Dealer Layout                                              │
│ ┌──────────────┐ ┌───────────────────────────────────────┐ │
│ │ Sidebar      │ │ 차량 등록                              │ │
│ │ - 내 차량     │ │ 기본 정보                              │ │
│ │ - 등록하기    │ │ [차량명] [제조사] [가격] [연식]        │ │
│ │ - 상담방      │ │ 차량 스펙                              │ │
│ │              │ │ [차종] [연료] [주행거리] [지역]        │ │
│ │              │ │ 사진 업로드                            │ │
│ │              │ │ [파일 선택] [미리보기]                 │ │
│ │              │ │ 설명                                  │ │
│ │              │ │ [textarea]                            │ │
│ │              │ │ [취소] [저장]                          │ │
│ └──────────────┘ └───────────────────────────────────────┘ │
└────────────────────────────────────────────────────────────┘
```

요구사항:

| 항목 | 설명 |
| --- | --- |
| 접근 권한 | `dealer`만 접근 |
| 사진 업로드 | `multipart/form-data`, `multer`, `/uploads` 사용 |
| 저장 필드 | 요구사항의 차량 데이터 구조 전체 반영 |
| 수정 화면 | 기존 차량 값을 폼에 채운 뒤 수정 |

| 이벤트 | 설명 |
| --- | --- |

| `leave-room` | 상담방 나가기 |
| `dealer-online` | 딜러 온라인 상태 |
| `dealer-offline` | 딜러 오프라인 상태 |

## 12. 관리자 대시보드

관리자 화면은 사이드바를 우선 사용한다. 필요하면 상세 섹션 내부에 상단 탭을 함께 사용한다.

```text
┌────────────────────────────────────────────────────────────┐
│ Admin Layout                                               │
│ ┌──────────────┐ ┌───────────────────────────────────────┐ │
│ │ Sidebar      │ │ Dashboard                             │ │
│ │ - 대시보드    │ │ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐  │ │
│ │ - 차량 관리   │ │ │ 차량 │ │ 사용자│ │ 상담방│ │ 메시지│  │ │
│ │ - 사용자 관리 │ │ └──────┘ └──────┘ └──────┘ └──────┘  │ │
│ │ - 상담 현황   │ │ 최근 상담                              │ │
│ │              │ │ table                                  │ │
│ └──────────────┘ └───────────────────────────────────────┘ │
└────────────────────────────────────────────────────────────┘
```

관리자 화면 구성:

| 화면 | 내용 |
| --- | --- |
| 대시보드 | 차량 수, 사용자 수, 상담방 수, 최근 메시지 요약 |
| 차량 관리 | 전체 차량 테이블, 상세 확인, 삭제 또는 상태 확인 |
| 사용자 관리 | 사용자 목록, 역할 확인 |
| 상담 현황 | 상담방 목록, 차량명, 구매자, 딜러, 마지막 메시지 |

## 17. UI 구현 기준

| 항목 | 기준 |
| --- | --- |
| 스타일 | 순수 Tailwind CSS |
| daisyUI | class 제거 후 패키지 제거 |
| 카드 radius | 8px 이하 권장 |
| 첫 화면 | 차량 목록/검색 중심 |
| 목록 | 일반 사용자는 카드형 |
| 관리 | 딜러/관리자는 테이블형 |
| 관리자 내비게이션 | 사이드바 우선 |
| 다크 모드 | 1차 제외 |
| 이미지 없음 | placeholder 사용 |

## 18. 구현 우선순위

| 순서 | 화면 | 이유 |
