# 와이어프레임

## 전체 화면 구성

React 화면은 자동차 CRUD를 중심으로 단순한 관리자형 구조로 구성한다.
Tailwind CSS와 daisyUI 컴포넌트를 사용해 빠르게 구현할 수 있는 형태를 기준으로 한다.

```text
┌────────────────────────────────────────────────────────────┐
│ Navbar                                                     │
│ [자동차 관리]                              [목록] [등록]   │
├────────────────────────────────────────────────────────────┤
│ Main Content                                               │
│ - 목록 / 등록 / 상세 / 수정 화면 중 하나 표시              │
└────────────────────────────────────────────────────────────┘
```

권장 React 컴포넌트 구조:

| 컴포넌트 | 역할 |
| --- | --- |
| `App` | 전체 라우팅 또는 화면 상태 관리 |
| `Layout` | 공통 navbar와 본문 영역 구성 |
| `CarList` | 자동차 목록, 검색, 필터 표시 |
| `CarForm` | 등록/수정 입력 폼 |
| `CarDetail` | 자동차 상세 정보 표시 |
| `DeleteConfirmModal` | 삭제 확인 모달 |
| `AlertMessage` | 성공/오류 메시지 표시 |

## 목록 화면 와이어프레임

```text
┌────────────────────────────────────────────────────────────┐
│ 자동차 목록                                      [등록하기] │
├────────────────────────────────────────────────────────────┤
│ [제조사 검색 input] [검색]  [최소가격] [최대가격] [필터]   │
├────────────────────────────────────────────────────────────┤
│ alert: 조회 성공 또는 오류 메시지                          │
├────────────────────────────────────────────────────────────┤
│ table                                                      │
│ ┌────┬────────┬──────────┬──────┬────────┬──────────────┐ │
│ │ ID │ 이름   │ 제조사   │ 연식 │ 가격   │ 관리         │ │
│ ├────┼────────┼──────────┼──────┼────────┼──────────────┤ │
│ │ 1  │ Sonata │ HYUNDAI  │ 2023 │ 2500   │ 상세 수정 삭제│ │
│ │ 2  │ K5     │ KIA      │ 2024 │ 2700   │ 상세 수정 삭제│ │
│ └────┴────────┴──────────┴──────┴────────┴──────────────┘ │
└────────────────────────────────────────────────────────────┘
```

목록 화면 요구사항:

| 항목 | 설명 |
| --- | --- |
| 등록하기 버튼 | 등록 화면으로 이동한다. |
| 제조사 검색 | `GET /cars/search?company=...` API를 호출한다. |
| 가격 필터 | `GET /cars/filter?minPrice=...&maxPrice=...` API를 호출한다. |
| 상세 버튼 | 상세 화면으로 이동한다. |
| 수정 버튼 | 수정 화면으로 이동한다. |
| 삭제 버튼 | 삭제 확인 모달을 연다. |

## 등록 화면 와이어프레임

```text
┌────────────────────────────────────────────────────────────┐
│ 자동차 등록                                                │
├────────────────────────────────────────────────────────────┤
│ 이름                                                       │
│ [input: Sonata]                                            │
│ 제조사                                                     │
│ [input: HYUNDAI]                                           │
│ 연식                                                       │
│ [input: 2023]                                              │
│ 가격                                                       │
│ [input: 2500]                                              │
├────────────────────────────────────────────────────────────┤
│ [취소]                                      [등록]          │
└────────────────────────────────────────────────────────────┘
```

등록 화면 요구사항:

| 항목 | 설명 |
| --- | --- |
| 입력 폼 | 이름, 제조사, 연식, 가격을 입력한다. |
| 등록 버튼 | 유효성 검사 후 `POST /cars` API를 호출한다. |
| 취소 버튼 | 목록 화면으로 돌아간다. |
| 오류 표시 | 입력값이 올바르지 않으면 alert로 안내한다. |

## 수정 화면 와이어프레임

```text
┌────────────────────────────────────────────────────────────┐
│ 자동차 수정                                                │
├────────────────────────────────────────────────────────────┤
│ 이름                                                       │
│ [input: 기존 자동차 이름]                                  │
│ 제조사                                                     │
│ [input: 기존 제조사]                                       │
│ 연식                                                       │
│ [input: 기존 연식]                                         │
│ 가격                                                       │
│ [input: 기존 가격]                                         │
├────────────────────────────────────────────────────────────┤
│ [취소]                                      [수정 저장]     │
└────────────────────────────────────────────────────────────┘
```

수정 화면 요구사항:

| 항목 | 설명 |
| --- | --- |
| 초기 데이터 | `GET /cars/:id`로 기존 자동차 정보를 불러온다. |
| 수정 저장 버튼 | 유효성 검사 후 `PUT /cars/:id` API를 호출한다. |
| 취소 버튼 | 상세 화면 또는 목록 화면으로 돌아간다. |

## 상세 화면 와이어프레임

```text
┌────────────────────────────────────────────────────────────┐
│ 자동차 상세                                                │
├────────────────────────────────────────────────────────────┤
│ card                                                       │
│ ┌────────────────────────────────────────────────────────┐ │
│ │ Sonata                                   badge HYUNDAI │ │
│ │ ID: 1                                                  │ │
│ │ 연식: 2023                                             │ │
│ │ 가격: 2500만원                                         │ │
│ └────────────────────────────────────────────────────────┘ │
├────────────────────────────────────────────────────────────┤
│ [목록으로]                         [수정] [삭제]           │
└────────────────────────────────────────────────────────────┘
```

상세 화면 요구사항:

| 항목 | 설명 |
| --- | --- |
| 상세 정보 | 이름, 제조사, 연식, 가격, ID를 표시한다. |
| 목록으로 버튼 | 목록 화면으로 이동한다. |
| 수정 버튼 | 수정 화면으로 이동한다. |
| 삭제 버튼 | 삭제 확인 모달을 연다. |

## 삭제 확인 모달 와이어프레임

```text
┌──────────────────────────────────────┐
│ modal                                │
│ 자동차 삭제                          │
├──────────────────────────────────────┤
│ Sonata 자동차를 삭제하시겠습니까?    │
│ 삭제한 데이터는 되돌릴 수 없습니다.  │
├──────────────────────────────────────┤
│ [취소]                    [삭제]     │
└──────────────────────────────────────┘
```

삭제 확인 모달 요구사항:

| 항목 | 설명 |
| --- | --- |
| 취소 버튼 | 모달을 닫고 데이터를 유지한다. |
| 삭제 버튼 | `DELETE /cars/:id` API를 호출한다. |
| 삭제 성공 | 목록 화면으로 이동하고 성공 alert를 표시한다. |

## 사용 예정 Tailwind CSS / daisyUI 컴포넌트 목록

| 목적 | daisyUI 컴포넌트 | Tailwind CSS 예시 |
| --- | --- | --- |
| 상단 메뉴 | `navbar` | `px-6`, `shadow-sm` |
| 본문 영역 | `container` 성격의 레이아웃 | `max-w-5xl`, `mx-auto`, `p-6` |
| 목록 표 | `table` | `table-zebra`, `w-full` |
| 입력 필드 | `input` | `input-bordered`, `w-full` |
| 버튼 | `button` | `btn`, `btn-primary`, `btn-error` |
| 상세 카드 | `card` | `bg-base-100`, `shadow` |
| 모달 | `modal` | `modal-box` |
| 메시지 | `alert` | `alert-success`, `alert-error` |
| 제조사 표시 | `badge` | `badge-outline`, `badge-info` |

## 화면 흐름

```text
[목록 화면]
   │
   ├─ 등록하기 클릭
   │    └─ [등록 화면] ─ 등록 성공 ─> [목록 화면]
   │
   ├─ 상세 클릭
   │    └─ [상세 화면]
   │         ├─ 수정 클릭 ─> [수정 화면] ─ 수정 성공 ─> [상세 화면 또는 목록 화면]
   │         └─ 삭제 클릭 ─> [삭제 확인 모달] ─ 삭제 성공 ─> [목록 화면]
   │
   ├─ 수정 클릭
   │    └─ [수정 화면] ─ 수정 성공 ─> [목록 화면]
   │
   └─ 삭제 클릭
        └─ [삭제 확인 모달] ─ 삭제 성공 ─> [목록 화면]
```
