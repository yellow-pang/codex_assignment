# 2026-06-05 7단계 UI 전면 개선 상세 설명

## 이 문서의 목표

이번 단계에서는 daisyUI 기반의 기존 UI를 순수 Tailwind CSS로 전환하고, Modern Car Marketplace 스타일로 전면 개선했다.

이 문서는 다음 내용을 처음 보는 사람이 이해할 수 있게 정리한다.

- 왜 daisyUI를 제거하고 순수 Tailwind CSS로 전환했는지
- 공통 스타일 클래스 패턴이 어떻게 구성되었는지
- 화면별로 어떤 방식으로 변경되었는지
- 신규 컴포넌트(CarCardGrid, ChatRoomList, ChatRoom)가 무엇을 담당하는지
- 서버에 어떤 API가 추가되었는지

## 한 줄 요약

전체 UI를 daisyUI 의존에서 벗어나 `@layer components` 기반의 공통 Tailwind 클래스 패턴으로 통일하고, 차량 목록을 카드 그리드로 전환하며 상담방 목록과 채팅 화면 UI 구조를 신규 구현했다.

## 작업 전 상태

6단계까지 서비스 핵심 기능(차량 CRUD, Firebase 인증, 역할 관리, 차량 상세 URL, 상담방 생성)이 완성된 상태였다.
그러나 UI 전체에 daisyUI 클래스(`btn`, `card`, `badge`, `table-zebra`, `modal`, `alert`, `tabs`, `stats`, `loading-spinner` 등)가 혼용되어 있었다.

## style.css — 공통 클래스 패턴

기존 방식은 컴포넌트마다 Tailwind 클래스를 직접 나열하는 구조였다.
변경 후에는 `@layer components`를 사용해 자주 반복되는 패턴을 하나의 이름으로 정의했다.

```css
@layer components {
  /* 버튼 */
  .c-btn        { @apply inline-flex items-center justify-center ... }
  .c-btn-primary  { @apply c-btn bg-blue-600 text-white hover:bg-blue-700; }
  .c-btn-outline  { @apply c-btn border border-gray-300 bg-white ...; }
  .c-btn-danger   { @apply c-btn bg-red-600 text-white hover:bg-red-700; }
  .c-btn-warning  { @apply c-btn bg-amber-500 text-white ...; }
  .c-btn-ghost    { @apply c-btn text-gray-600 hover:bg-gray-100; }

  /* 입력 필드 */
  .c-input        { @apply block w-full rounded-md border border-gray-300 ...; }
  .c-select       { ... }
  .c-textarea     { ... }

  /* 카드 */
  .c-card         { @apply rounded-xl border border-gray-200 bg-white shadow-sm; }

  /* 배지 */
  .c-badge-blue   { @apply ... bg-blue-50 text-blue-700; }
  .c-badge-gray   { @apply ... bg-gray-100 text-gray-600; }
  ...

  /* 상태 배너 */
  .c-alert-error   { @apply ... border-red-200 bg-red-50 text-red-700; }
  .c-alert-success { @apply ... border-green-200 bg-green-50 text-green-700; }
  ...
}
```

이렇게 하면 컴포넌트에서는 `className="c-btn-primary"` 한 줄로 일관된 버튼 스타일을 사용할 수 있다.

## 신규 컴포넌트: ChatRoomList

`/chats` 라우트에서 렌더링된다.

- 로그인한 사용자의 uid로 `GET /api/chats/rooms?uid=xxx`를 호출한다.
- 상담방 목록을 차량 썸네일 + 차량명 + 상대방 이름 + 시간 형태로 나열한다.
- 딜러는 구매자 이름, 구매자는 딜러 이름을 표시한다.
- 목록 클릭 시 `/chats/:roomId`로 이동한다.
- 상담 내역이 없으면 안내 메시지를 표시한다.

```jsx
const isDealer = userProfile?.role === "dealer";
const otherPartyName = isDealer ? room.buyerName : room.dealerName;
```

## 신규 컴포넌트: ChatRoom

`/chats/:roomId` 라우트에서 렌더링된다.

- `GET /api/chats/rooms/:roomId/messages`로 이전 메시지를 조회한다.
- 내 메시지는 우측 파란 말풍선, 상대 메시지는 좌측 회색 말풍선으로 구분한다.
- 메시지 추가 시 자동으로 최하단 스크롤한다.
- 메시지 전송 버튼은 UI만 구성되어 있으며, 실제 Socket.io 연결은 8단계에서 완성한다.

```jsx
// 내 메시지 (우측, 파란 말풍선)
<div className="rounded-2xl rounded-br-sm bg-blue-600 text-white ...">
// 상대 메시지 (좌측, 회색 말풍선)
<div className="rounded-2xl rounded-bl-sm bg-gray-100 text-gray-900 ...">
```

## 서버 변경 내용

### 신규 API: GET /api/chats/rooms

내 uid 기준 상담방 목록을 반환한다.

```
GET /api/chats/rooms?uid=firebase-user-uid
```

- `buyerId` 또는 `dealerId`가 uid와 일치하는 상담방을 `updatedAt` 내림차순으로 반환한다.
- uid가 없으면 `400`을 응답한다.

### 신규 API: GET /api/chats/rooms/:roomId/messages

특정 상담방의 이전 메시지를 반환한다.

```
GET /api/chats/rooms/:roomId/messages
```

- `messages` 컬렉션에서 `roomId`가 일치하는 메시지를 `createdAt` 오름차순으로 반환한다.
- roomId가 없으면 `400`을 응답한다.
- 8단계 Socket.io 연결 후 실시간 메시지 저장과 조회가 완성된다.

```js
function getMessagesCollection() {
  return getCollection("messages");
}
```

## daisyUI 제거 과정

1. 모든 컴포넌트에서 daisyUI 클래스를 `c-*` 패턴 또는 직접 Tailwind 클래스로 교체했다.
2. `frontend/tailwind.config.js`에서 `import daisyui from "daisyui"` 및 `plugins: [daisyui]`, `daisyui: { themes: [...] }` 설정을 제거했다.
3. `npm uninstall daisyui`로 패키지를 제거했다.

제거된 daisyUI 클래스 목록:

| 제거된 class                                                                 | 대체 방향                                                                        |
| ---------------------------------------------------------------------------- | -------------------------------------------------------------------------------- |
| `btn`, `btn-primary`, `btn-ghost`, `btn-outline`, `btn-error`, `btn-warning` | `c-btn-primary`, `c-btn-outline`, `c-btn-danger`, `c-btn-warning`, `c-btn-ghost` |
| `card`, `card-body`, `card-title`, `card-actions`                            | `c-card`, 직접 `div + p + flex`                                                  |
| `input input-bordered`                                                       | `c-input`                                                                        |
| `select select-bordered`                                                     | `c-select`                                                                       |
| `textarea textarea-bordered`                                                 | `c-textarea`                                                                     |
| `file-input file-input-bordered`                                             | Tailwind `file:` 유사 선택자 직접 작성                                           |
| `badge`, `badge-info`, `badge-outline`                                       | `c-badge-blue`, `c-badge-gray` 등                                                |
| `navbar`                                                                     | `sticky top-0 border-b bg-white`                                                 |
| `stats`, `stat`, `stat-title`, `stat-value`                                  | `c-card + SummaryCard` 컴포넌트                                                  |
