# 2026-06-06 12단계 상담·관리자·모바일 UI 고도화 상세 설명

## 이 문서의 목표

이번 단계에서는 11단계 Modern UI 재설계에서 2차 개선으로 남겨 둔 상담방 목록, 채팅 화면, 딜러 관리, 관리자 대시보드, 모바일 하단 내비게이션을 고도화했다.

기능 구조를 새로 만들기보다 기존 `/api/cars`, `/api/users`, `/api/chats`, Firebase Authentication, Socket.io 이벤트 구조를 유지하면서 화면 사용성과 정보 구조를 정리하는 데 집중했다.

## 한 줄 요약

Clean Blue Trust 색상 체계를 유지하면서 상담 화면은 밝은 실시간 상담 UI로, 딜러/관리자 화면은 slate/navy 업무형 화면으로 개선하고 모바일 하단 내비게이션을 추가했다.

## 주요 변경 내용

### 상담방 목록

- 상단에 밝은 gradient 기반 상담 Hero를 추가했다.
- 전체 상담 수와 최근 대화 수 요약을 표시한다.
- 상담방 검색 입력을 추가해 차량명, 구매자명, 딜러명, 마지막 메시지 기준으로 필터링한다.
- 상담방 카드는 차량 썸네일, 차량명, 상대방, 마지막 메시지, 최근 시간, 상태 배지를 중심으로 재구성했다.
- 목록이 비어 있거나 검색 결과가 없으면 차량 목록으로 이동하는 CTA를 제공한다.

### 실시간 채팅 화면

- 데스크톱에서는 좌측 차량 상담 정보 패널과 우측 채팅 영역으로 나눴다.
- 모바일에서는 차량 썸네일과 차량명, 상대방, 딜러 온라인 상태를 상단에 압축 표시한다.
- Socket.io 연결 상태를 별도 안내바로 표시한다.
- 내 메시지는 `blue-600`, 상대 메시지는 `slate-100` 말풍선으로 구분했다.
- 채팅 입력창은 모바일 하단 내비게이션과 겹치지 않도록 화면 높이와 본문 여백을 조정했다.

### 딜러 관리

- `/dealer` 라우트를 추가했다.
- `DealerDashboard.jsx`를 추가해 현재 차량 목록 중 `dealerId`가 로그인한 딜러 UID와 같은 차량만 표시한다.
- 등록 차량 수, 총 판매가, 판매 지역 수를 요약 카드로 제공한다.
- 데스크톱은 테이블형, 모바일은 카드형으로 내 차량 목록을 보여준다.
- 상세, 수정, 삭제 버튼은 기존 핸들러를 그대로 사용한다.

### 관리자 대시보드

- `AdminUserPanel.jsx`를 `slate-950` 사이드바 + white content card 구조로 재구성했다.
- 사용자 관리, 차량 관리, 상담 현황 메뉴를 사이드바로 분리했다.
- 사용자 관리 기능의 승인, 거절, 딜러 지정, 딜러 회수, admin 지정/해제 버튼은 유지했다.
- 차량 관리와 상담 현황은 새 관리자 API가 필요한 범위를 무리하게 구현하지 않고 준비 중 패널로 표시했다.

### 모바일 하단 내비게이션

- `MobileBottomNav.jsx`를 추가했다.
- 모바일에서만 표시되며 차량 검색, 내 상담, 내 차량, 관리자 탭을 권한에 따라 노출한다.
- 데스크톱은 기존 Header 내비게이션을 유지한다.
- 본문 하단 여백을 추가해 하단 탭과 콘텐츠가 겹치지 않도록 했다.
