# 20단계 제출용 개발 문서 작성

## 1. 작업 목적

프로젝트 제출 시 요구사항 정의서 외에 함께 제시할 수 있는 개발 문서 묶음을 작성했다.

기존 `plans`, `steps`, `pr` 문서는 실제 개발 진행 기록에 가깝기 때문에, 이번 단계에서는 평가자나 발표자가 한 번에 읽기 쉬운 별도 제출용 문서로 정리했다.

## 2. 작업 요약

| 구분 | 내용 |
| --- | --- |
| 문서 범위 | 프로젝트 개요, 시스템 설계, DB/API 명세, 화면 흐름, 검증/배포 정리 |
| 작성 위치 | `docs/` |
| 작성 형식 | Markdown |
| 문서 스타일 | 표, Mermaid 흐름도, 단계별 설명 중심 |
| 기능 영향 | 없음 |

## 3. 생성한 파일

```text
docs/개발문서_00_문서목록.md
docs/개발문서_01_프로젝트_개요.md
docs/개발문서_02_시스템_설계서.md
docs/개발문서_03_DB_API_명세서.md
docs/개발문서_04_화면_흐름_설계서.md
docs/개발문서_05_개발_검증_배포_정리.md
docs/steps/2026-06-07-20-submission-development-documents.md
docs/pr/2026-06-07-20-submission-development-documents-pr.md
```

## 4. 문서별 작성 내용

| 문서 | 주요 내용 |
| --- | --- |
| `개발문서_00_문서목록.md` | 제출 문서 구성과 PDF 변환 순서 |
| `개발문서_01_프로젝트_개요.md` | 프로젝트 목적, 사용자 유형, 핵심 기능, 기술 스택 |
| `개발문서_02_시스템_설계서.md` | 전체 아키텍처, 요청 흐름, 폴더 구조, 보안 설계 |
| `개발문서_03_DB_API_명세서.md` | MongoDB 컬렉션, REST API, Socket.io 이벤트 |
| `개발문서_04_화면_흐름_설계서.md` | 사용자별 화면 흐름, 주요 화면 구성, 반응형 기준 |
| `개발문서_05_개발_검증_배포_정리.md` | 개발 단계, 검증 명령, 배포 구조, 최종 점검표 |

## 5. 작성 기준

- 최신 요구사항 문서와 실제 코드 구조를 함께 기준으로 삼았다.
- 초기 CRUD 요구사항과 최종 구현 범위가 다른 부분은 프로젝트 확장 과정으로 설명했다.
- 민감한 환경변수 값, Firebase Secret, MongoDB 접속 문자열은 포함하지 않았다.
- PDF 변환 시 표와 흐름도를 그대로 사용할 수 있도록 Markdown 표와 Mermaid 문법을 사용했다.

## 6. 검증 결과

| 검증 | 결과 |
| --- | --- |
| 신규 문서 생성 여부 확인 | 성공 |
| 기능 코드 변경 여부 확인 | 기능 코드 변경 없음 |
| 빌드 실행 | 문서 작업만 진행해 실행하지 않음 |

## 7. 남은 작업

1. PDF 변환 도구가 Mermaid를 지원하지 않는 경우, 흐름도는 이미지로 변환해 삽입한다.
2. 제출 양식에서 문서 제목 규칙이 따로 있으면 파일명만 조정한다.
3. 실제 배포 URL이 확정되면 `개발문서_05_개발_검증_배포_정리.md`에 추가할 수 있다.
