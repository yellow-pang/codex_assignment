# 요구사항 정의서

## 프로젝트 개요

이 프로젝트는 Express 기반 자동차 REST API와 React 프론트엔드를 통합한 과제용 CRUD 웹 애플리케이션이다.
사용자는 자동차 목록을 조회하고, 자동차 정보를 등록, 상세 조회, 수정, 삭제할 수 있다.

## 개발 목적

- Express REST API의 CRUD 구조를 이해한다.
- React에서 API 데이터를 호출하고 화면에 표시하는 방법을 익힌다.
- Tailwind CSS와 daisyUI를 활용해 기본적인 관리자형 UI를 구성한다.
- Render 클라우드 배포 흐름을 이해한다.
- GitHub Actions를 이용한 CI/CD 기본 흐름을 구성한다.

## 사용 기술

| 구분 | 기술 |

| 백엔드 | Node.js, Express |
| 프론트엔드 | React.js, Vite |

## 주요 기능

| 기능 | 설명 |
| --- | --- |
| 자동차 목록 조회 | 등록된 자동차 전체 목록을 표 형태로 보여준다. |
| 자동차 상세 조회 | 선택한 자동차의 상세 정보를 보여준다. |
| 자동차 등록 | 이름, 제조사, 연식, 가격을 입력해 새 자동차를 등록한다. |
| 자동차 수정 | 기존 자동차 정보를 수정한다. |
| 자동차 삭제 | 삭제 확인 후 자동차를 목록에서 제거한다. |
| 제조사 검색 | 제조사명으로 자동차 목록을 필터링한다. |
| 가격 필터 | 최소 가격, 최대 가격 기준으로 자동차 목록을 필터링한다. |

## CRUD 기능 요구사항

| 구분 | 사용자 동작 | 처리 내용 |
| --- | --- | --- |
| Create | 등록 화면에서 자동차 정보 입력 후 저장 | `POST /cars` 요청으로 새 자동차를 추가한다. |
| Read | 목록 화면 진입 | `GET /cars` 요청으로 전체 목록을 조회한다. |
| Read | 상세 버튼 클릭 | `GET /cars/:id` 요청으로 단일 자동차를 조회한다. |
| Update | 수정 화면에서 정보 변경 후 저장 | `PUT /cars/:id` 요청으로 자동차 정보를 수정한다. |
| Delete | 삭제 버튼 클릭 후 확인 | `DELETE /cars/:id` 요청으로 자동차를 삭제한다. |

## API 요구사항

기존 API 경로는 유지한다.
React 개발 서버에서는 Vite 프록시를 이용해 `/api` 요청을 Express 서버로 전달한다.

| Method | Express API | React 호출 경로 | 설명 |
| --- | --- | --- | --- |
| GET | `/cars` | `/api/cars` | 자동차 전체 목록 조회 |
| GET | `/cars/:id` | `/api/cars/:id` | 자동차 상세 조회 |
| POST | `/cars` | `/api/cars` | 자동차 등록 |
| PUT | `/cars/:id` | `/api/cars/:id` | 자동차 수정 |
| DELETE | `/cars/:id` | `/api/cars/:id` | 자동차 삭제 |
| GET | `/cars/search?company=HYUNDAI` | `/api/cars/search?company=HYUNDAI` | 제조사 검색 |
| GET | `/cars/filter?minPrice=2000&maxPrice=3000` | `/api/cars/filter?minPrice=2000&maxPrice=3000` | 가격 필터 |

### 자동차 데이터 구조

| 필드 | 타입 | 필수 여부 | 설명 | 예시 |
| --- | --- | --- | --- | --- |
| `_id` | number | 필수 | 자동차 고유 ID | `1` |
| `name` | string | 필수 | 자동차 이름 | `Sonata` |
| `company` | string | 필수 | 제조사 | `HYUNDAI` |
| `year` | number | 필수 | 연식 | `2023` |
| `price` | number | 필수 | 가격, 단위는 만원 | `2500` |

### API 응답 기준

| 상황 | 상태 코드 | 응답 |
| --- | --- | --- |
| 목록 조회 성공 | `200` | 자동차 배열 |
| 상세 조회 성공 | `200` | 자동차 객체 |
| 등록 성공 | `201` | 등록된 자동차 객체 |
| 수정 성공 | `200` | 수정된 자동차 객체 |
| 삭제 성공 | `200` | 삭제된 자동차 객체 |
| 없는 ID 조회, 수정, 삭제 | `404` | `{ "message": "자동차를 찾을 수 없습니다." }` |

## 화면 요구사항

| 화면 | 주요 내용 |
| --- | --- |
| 목록 화면 | 자동차 목록 표, 등록 버튼, 검색 입력, 가격 필터, 상세/수정/삭제 버튼 |
| 등록 화면 | 자동차 이름, 제조사, 연식, 가격 입력 폼 |

| 삭제 확인 모달 | 삭제 대상 이름 표시, 취소/삭제 버튼 |

## 유효성 검사 요구사항

| 항목 | 검사 기준 | 오류 메시지 예시 |
| --- | --- | --- |
| 자동차 이름 | 빈 값 불가 | 자동차 이름을 입력해주세요. |

| 가격 필터 | 입력 시 숫자만 허용 | 가격은 숫자로 입력해주세요. |

## 배포 요구사항

| 항목 | 내용 |
| --- | --- |
| 배포 서비스 | Render |
| 배포 대상 | Express API 서버와 React 빌드 결과물 |
| 빌드 명령 | 프론트엔드 의존성 설치 후 Vite 빌드 |
| 실행 명령 | Express 서버 실행 |
| 환경 변수 | 필요 시 `PORT` 사용 |

Render 배포 시 Express 서버가 React 정적 파일을 제공하도록 구성할 수 있다.
과제 범위에서는 Render에서 서비스가 정상 실행되고 자동차 목록 화면에 접근 가능한 것을 목표로 한다.

## CI/CD 요구사항

| 항목 | 내용 |
| --- | --- |
| 도구 | GitHub Actions |
