# PR: 제출용 개발 문서 작성

## PR 제목

```text
docs: 제출용 개발 문서 세트 추가
```

## 작업 배경

요구사항 정의서는 별도로 준비되어 있으나, 프로젝트 제출 시 개발 구조와 구현 결과를 설명할 수 있는 문서가 필요했다.

기존 `docs/plans`, `docs/steps`, `docs/pr`는 작업 진행 기록으로는 충분하지만, 제출 자료로 바로 묶기에는 흐름이 분산되어 있었다. 이번 작업에서는 PDF 변환을 전제로 읽기 쉬운 개발 문서 세트를 새로 작성했다.

## 변경 내용

### 1) 문서 목록 추가

- 제출 문서의 전체 구성을 확인할 수 있는 `개발문서_00_문서목록.md`를 추가했다.
- PDF 변환 순서와 문서별 목적을 정리했다.

### 2) 프로젝트 개요 문서 추가

- 프로젝트 목적, 사용자 유형, 핵심 기능, 기술 스택을 정리했다.
- 초기 CRUD 과제에서 실시간 마켓 서비스로 확장된 흐름을 설명했다.

### 3) 시스템 설계 문서 추가

- React, Express, MongoDB Atlas, Firebase, Socket.io의 연결 구조를 정리했다.
- REST API 요청 흐름과 실시간 상담 흐름을 Mermaid 다이어그램으로 추가했다.

### 4) DB/API 명세 문서 추가

- `cars`, `users`, `chat_rooms`, `messages`, `dealer_presence` 컬렉션 구조를 표로 정리했다.
- 차량, 사용자, 상담 API와 Socket.io 이벤트를 명세했다.

### 5) 화면 흐름과 검증/배포 문서 추가

- 일반 사용자, 딜러, 관리자 화면 흐름을 구분해 정리했다.
- 개발 단계, 검증 명령, Render 배포 구조, 제출 전 점검표를 추가했다.
