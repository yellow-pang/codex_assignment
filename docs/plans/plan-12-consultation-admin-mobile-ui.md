# 12단계 상담·관리자·모바일 UI 고도화 계획

## 1. 문서 목적

11단계 Modern UI 재설계에서 제외한 2차 개선 범위인 상담방 목록, 실시간 채팅 화면, 딜러 관리, 관리자 대시보드, 모바일 하단 내비게이션 고도화 방향을 정리한다.

이번 작업은 이미 구현된 API, Firebase Authentication, MongoDB, Socket.io 이벤트 구조를 유지하고 프론트엔드 UI 표현 계층을 개선하는 것을 목표로 한다.

## 4. 현재 UI 상태 요약

| 화면              | 현재 상태                                          | 개선 필요점                                                           |
| ----------------- | -------------------------------------------------- | --------------------------------------------------------------------- |
| 상담방 목록       | 상담방 리스트, 차량 썸네일, 상대방, 갱신 시간 표시 | 검색/상태 필터, 마지막 메시지, 빈 상태, 모바일 리스트 밀도 개선       |
| 채팅 화면         | 상단 차량명, 딜러 상태, 메시지 말풍선, 입력창 구성 | 차량 정보 패널, 연결 상태 안내, 메시지 가독성, 모바일 높이 처리 개선  |
| 딜러 관리         | 차량 등록/수정 폼은 11단계에서 개선됨              | 딜러가 보는 내 차량 관리 목록 또는 관리 진입 UX 정리 필요             |
| 관리자 대시보드   | 사용자 관리와 요약 카드 구현, 차량/상담 탭 비활성  | 사이드바형 관리 화면, 사용자/딜러 승인/상담 현황을 더 업무형으로 정리 |
| 모바일 내비게이션 | 상단 햄버거 메뉴 제공                              | 주요 화면을 빠르게 이동하는 하단 탭 추가 검토                         |

## 5. 구현 목표

1. 상담방 목록을 실시간 상담 서비스처럼 보이도록 고도화한다.
2. 채팅 화면을 차량 상담 맥락이 잘 보이는 모바일 친화형 대화 UI로 개선한다.
3. 딜러 관리 화면은 승인된 딜러가 자신의 등록 차량과 상담 진입을 쉽게 확인하는 방향으로 정리한다.
4. 관리자 화면은 sidebar 또는 split layout 기반의 업무형 대시보드로 개선한다.
5. 모바일 375px 기준에서 주요 화면 접근성을 높이기 위해 하단 내비게이션을 추가한다.
6. 기존 API 경로, Socket.io 이벤트 이름, Firebase/MongoDB 데이터 구조는 변경하지 않는다.

## 6. 화면별 작업 방향

### 6.1 상담방 목록

| 항목        | 작업 방향                                                                         |
| ----------- | --------------------------------------------------------------------------------- |
| 레이아웃    | 상단 요약 영역 + 검색 입력 + 상담방 리스트 구조                                   |
| 리스트 카드 | 차량 썸네일, 차량명, 상대방 이름, 마지막 메시지, 최근 상담 시간, 온라인 상태 표시 |
| 사용자 역할 | 구매자는 딜러명 중심, 딜러는 구매자명 중심으로 상대방을 표시                      |
| 빈 상태     | 상담 내역이 없을 때 차량 목록으로 이동하는 CTA 제공                               |
| 모바일      | 한 줄이 너무 길어지지 않도록 썸네일과 텍스트를 압축하고 터치 영역 확보            |

### 6.2 채팅 화면

| 항목        | 작업 방향                                                          |
| ----------- | ------------------------------------------------------------------ |
| 상단 영역   | 차량명, 딜러명, 온라인 상태, 목록으로 돌아가기 버튼 정리           |
| 메시지 영역 | 내 메시지와 상대 메시지 색상/정렬 유지, 말풍선 폭과 시간 표시 개선 |
| 차량 맥락   | 데스크톱에서는 차량 정보 패널 또는 상담 요약 패널을 함께 표시      |
| 입력창      | 모바일에서 하단에 안정적으로 붙고 버튼/입력창이 겹치지 않게 조정   |
| 연결 상태   | Socket.io 연결 중, 연결 끊김, 오류 상태를 안내바로 분리            |

### 6.3 딜러 관리

| 항목 | 작업 방향                                                |
| ---- | -------------------------------------------------------- |
| 범위 | 신규 API 없이 현재 차량 목록 데이터와 권한 체크를 활용   |
| 화면 | 딜러가 등록한 차량 중심 관리 섹션을 구성                 |
| 기능 | 수정, 삭제, 상담 이동 버튼은 기존 핸들러를 유지          |
| 표현 | 관리자와 다르게 딜러 업무용 테이블/카드 혼합 형태로 정리 |
| 제한 | 딜러 승인 정책이나 MongoDB 컬렉션 구조는 변경하지 않음   |

### 6.4 관리자 대시보드

| 항목        | 작업 방향                                                         |
| ----------- | ----------------------------------------------------------------- |
| 레이아웃    | slate/navy 사이드바 + 본문 패널 구조                              |
| 사용자 관리 | 기존 사용자 목록, 딜러 승인, 권한 변경 기능 유지                  |
| 요약 카드   | 전체 사용자, 승인 대기, 딜러, 관리자 수를 더 읽기 쉽게 정리       |
| 상담 현황   | 기존 API에서 가능한 범위만 연결하고, 부족하면 준비 중 상태로 표시 |
| 차량 관리   | 실제 연결 범위가 불충분하면 비활성/준비 중으로 명확히 표시        |

### 6.5 모바일 하단 내비게이션

| 항목      | 작업 방향                                                               |
| --------- | ----------------------------------------------------------------------- |
| 노출 기준 | 모바일 화면에서만 표시                                                  |
| 기본 탭   | 차량 검색, 내 상담                                                      |
| 조건부 탭 | 딜러는 차량 등록 또는 내 차량, 관리자는 관리자 탭 표시                  |
| 인증 상태 | 비로그인 사용자는 로그인/회원가입 진입을 방해하지 않도록 최소 탭만 표시 |
| 접근성    | 버튼 크기와 라벨을 충분히 두고 현재 화면 active 상태 표시               |

## 7. 변경하지 않는 항목

| 항목                  | 이유                                                                                                            |
| --------------------- | --------------------------------------------------------------------------------------------------------------- |
| Express API 경로      | 기존 `/api/cars`, `/api/users`, `/api/chats` 구조 유지                                                          |
| Socket.io 이벤트 이름 | 요구사항의 `join-room`, `send-message`, `receive-message`, `leave-room`, `dealer-online`, `dealer-offline` 유지 |
| Firebase 설정         | 인증 흐름 변경 작업이 아님                                                                                      |
| MongoDB 컬렉션 구조   | UI 고도화 작업이므로 데이터 구조 변경 불필요                                                                    |
| Render 배포 구조      | 단일 Web Service 배포 기준 유지                                                                                 |
| 외부 이미지 스토리지  | 기존 `/uploads` 정적 경로 유지                                                                                  |
| 새 UI 라이브러리      | 순수 Tailwind CSS 기준 유지                                                                                     |

| 확인 항목              | 확정 방향                                                                                           |
| ---------------------- | --------------------------------------------------------------------------------------------------- |
| 작업 범위              | 상담방 목록, 채팅 화면, 딜러 관리, 관리자 대시보드, 모바일 하단 내비게이션을 모두 12단계에 포함     |

| 관리자 상담 현황       | 기존 API로 가능한 범위만 표시하고, 부족한 통계는 준비 중 상태로 둠                                  |

## 10. 작업 순서

1. 사용자 확인 사항을 확정한다.
2. 필요한 경우 `DealerDashboard.jsx`, `MobileBottomNav.jsx` 신규 컴포넌트를 만든다.
3. 상담방 목록 UI를 먼저 개선한다.
4. 채팅 화면 UI와 모바일 높이 처리를 개선한다.
5. 딜러 관리 화면을 현재 데이터와 권한 기준으로 구성한다.
6. 관리자 대시보드를 sidebar/summary/table 구조로 개선한다.
7. 모바일 하단 내비게이션을 추가하고 Header와 중복되지 않게 정리한다.
8. 375px, 768px, 1280px 기준으로 레이아웃을 점검한다.
9. daisyUI 계열 class가 재도입되지 않았는지 검색한다.
10. `npm.cmd --prefix frontend run build`와 `npm.cmd run build`를 실행한다.
11. 구현 완료 후 Step, PR, progress 문서를 갱신한다.

| 상담 목록   | 로그인 사용자 기준 `/chats` 화면 렌더링과 빈 상태 표시 확인                                                                |
| 채팅 화면   | `/chats/:roomId` 화면에서 메시지 목록, 입력창, 연결 상태 UI 확인                                                           |
| 딜러 화면   | 승인된 딜러 기준 내 차량 관리 진입과 기존 수정/삭제 흐름 유지 확인                                                         |
| 관리자 화면 | admin 기준 `/admin` 접근, 사용자 관리 버튼, 승인/거절 버튼 유지 확인                                                       |
| 모바일      | 375px 기준 하단 내비게이션이 본문/입력창과 겹치지 않음                                                                     |

## 13. 이번 계획의 결론

12단계는 11단계에서 완성한 사용자 중심 Modern Marketplace UI를 상담, 딜러 업무, 관리자 업무, 모바일 이동 경험까지 확장하는 작업이다.

기능 구조를 새로 만들기보다 이미 구현된 상담방, 메시지, 사용자 권한, 차량 관리 흐름을 더 쉽게 보고 사용할 수 있게 만드는 데 집중한다.
