# 18단계 프리미엄 마켓플레이스 UI 리프레시 계획

## 1. 문서 목적

이번 18단계는 사용자 피드백을 반영해 첫 화면의 프리미엄 인상, 구매 구간의 실용성, 운영 화면의 업무 효율을 동시에 확보하는 UI 리프레시 기준을 확정하기 위한 계획 문서다.

이 단계는 디자인 기준과 토큰을 확정하는 문서 작업이며, 코드 수정은 사용자 확인 후 진행한다.

## 4. 확정 디자인 방향

### 4.1 화면 성격 분리

1. 첫 페이지: 프리미엄 딜러십 스타일
2. 구매/검색/상세 구간: 실용형 마켓 스타일
3. 관리자/딜러 대시보드: 운영형 대시보드 스타일

### 4.2 통일 원칙

- 화면 성격은 다르게 가져가되, 색/타이포/컴포넌트 규칙은 하나의 토큰 체계로 통일한다.
- 첫 화면에서만 감성 표현을 강화하고, 나머지 화면은 가독성과 전환 효율을 우선한다.
- 기존 코발트 계열의 기본 Tailwind 느낌을 줄이고, 더 모던한 브랜드 톤으로 전환한다.

## 5. 모던 컬러 시스템 (코발트 대체)

다음 색상은 이번 단계의 전역 디자인 토큰이다.

| 토큰            | HEX       | 용도                                      |
| --------------- | --------- | ----------------------------------------- |
| `--brand-ink`   | `#0E1420` | 헤더, 주요 텍스트, 프리미엄 배경의 기준색 |
| `--brand-deep`  | `#14263D` | Hero 그라디언트 시작, 사이드바 배경       |
| `--brand-ocean` | `#1C4E6D` | Hero 그라디언트 중간톤, 강조 라인         |
| `--brand-mint`  | `#2FAE9B` | 핵심 CTA, 활성 상태, 링크 포인트          |
| `--brand-amber` | `#D98A3A` | 가격/핵심 수치 강조, 보조 CTA             |
| `--surface-0`   | `#F5F7FA` | 앱 기본 배경                              |
| `--surface-1`   | `#FFFFFF` | 카드/패널 배경                            |
| `--line-soft`   | `#D6DEE8` | 경계선, 입력창 기본 테두리                |
| `--text-main`   | `#172334` | 본문 텍스트                               |
| `--text-muted`  | `#5D6B7C` | 보조 텍스트                               |

### 5.1 사용 비율 가이드

- `brand-ink/deep/ocean`: 20%
- `surface-0/surface-1`: 70%
- `brand-mint/amber`: 10%

주의:

- 버튼, 배지, 링크, 상태 칩에서만 포인트 컬러를 사용한다.
- 화면 전체를 강한 색으로 채우지 않고, 대비는 명도 차와 여백으로 만든다.

## 6. 타이포/형태 시스템

### 6.1 타이포 제안

- 기본: `Pretendard`, `SUIT`, `sans-serif`
- 영문 강조(선택): `Plus Jakarta Sans`, `Pretendard`, `sans-serif`

### 6.2 타이포 스케일

| 용도        | 크기/두께         |
| ----------- | ----------------- |
| Hero 타이틀 | 40~52px / 700     |
| 섹션 타이틀 | 24~32px / 700     |
| 카드 타이틀 | 18~22px / 600     |
| 본문        | 14~16px / 400~500 |
| 캡션/메타   | 12~13px / 500     |

### 6.3 형태 규칙

- 카드 radius: 14px
- 입력/버튼 radius: 10px
- 그림자: 가벼운 단일 shadow 중심 (무거운 다중 shadow 지양)
- border: `1px solid var(--line-soft)` 기본

## 7. 화면별 스타일 적용 기준

### 7.1 첫 페이지 (프리미엄)

- 상단 Hero: `brand-deep -> brand-ocean` 그라디언트
- 카피: 짧고 강한 메시지 + 검색 진입 CTA 1개
- 카드 밀도: 낮게, 대표 차량 중심 큐레이션

### 7.2 차량 목록/검색/상세 (실용형)

- 배경: `surface-0`
- 검색 패널: 고정형/접이식 혼합, 빠른 필터 조작 우선
- 카드 정보 우선순위: 가격 > 연식/주행거리 > 지역 > 제조사/연료
- 상세 페이지: 정보 레이아웃 단순화 + 상담 CTA 고정

### 7.3 관리자/딜러 대시보드 (운영형)

- 사이드바: `brand-deep`
- 본문: `surface-0` + `surface-1` 카드
- 상태 색: 성공/주의/오류를 브랜드 포인트 기반으로 절제 사용
- 데이터 표: 강조색보다 정렬/간격/상태칩으로 가독성 확보

## 8. 컴포넌트 공통 규칙

1. 버튼

- Primary: `brand-mint` 채움
- Secondary: `surface-1` + `line-soft` border
- Danger: 저채도 레드 1종만 사용

2. 입력

- 기본 border `line-soft`, focus ring은 `brand-mint` 30% 투명도
- placeholder는 `text-muted`보다 연하게 유지

3. 배지/칩

- 상태칩은 최대 4색만 허용 (기본/성공/주의/오류)

4. 모달

- 오버레이는 불투명 검정보다 청색 기미가 있는 저채도 딤 처리

## 9. 반응형 기준

- 모바일 기준 시작점: 375px
- 주요 브레이크포인트: 640 / 768 / 1024 / 1280
- 모바일 우선 규칙:
  - 검색 패널은 1열
  - CTA는 폭 100% 우선
  - 카드 내부 정보는 2줄 초과 시 축약

## 10. 구현 순서 제안

1. 전역 토큰(색상/타이포/간격/radius) 정의
