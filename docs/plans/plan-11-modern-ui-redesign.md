## 1. 문서 목적

10단계 README 제출용 정리가 완료된 뒤, 고도화 첫 단계로 UI를 다시 현대적인 Car Market 서비스 화면으로 개선하기 위한 계획을 정리한다.

이번 계획은 이전 7단계 UI 전면 개선 결과가 `docs/images/확장_계획_디자인.png`와 다르게 보인 이유를 먼저 분석하고, 다음 구현에서 어떤 기준으로 화면을 다시 설계할지 확정하기 위해 작성한다.

## 4. 기대 디자인과 실제 구현 차이

### 4.1 목표 디자인의 특징

`docs/images/확장_계획_디자인.png`의 목표 화면은 다음 특징을 가진다.

| 영역        | 목표 특징                                                         |
| ----------- | ----------------------------------------------------------------- |
| 전체 톤     | 밝은 흰색 기반, 얇은 경계선, 부드러운 그림자, 넓은 여백           |
| 메인 Hero   | 차량 이미지와 검색 패널이 한 화면에서 자연스럽게 결합             |
| 검색 UI     | 라벨이 있는 입력 그룹, 둥근 흰색 패널, 파란색 CTA                 |
| 차량 카드   | 실제 차량 사진 중심, 가격 강조, 작은 스펙 아이콘, 하단 CTA 분리   |
| 상세 화면   | 큰 이미지 갤러리, 가격과 핵심 스펙 강조, 딜러 정보 카드           |
| 인증 화면   | 좌측 폼 + 우측 차량 이미지/브랜드 패널 또는 넓은 인증 카드        |
| 모바일      | 하단 내비게이션 느낌, 좁은 화면에서도 카드와 검색이 압축되지 않음 |
| 관리자 화면 | 짙은 사이드바, 대시보드 카드, 테이블 중심의 업무형 레이아웃       |

### 4.2 현재 구현의 특징

현재 7단계 결과는 다음 특징을 가진다.

| 영역        | 현재 상태                                                                   |
| ----------- | --------------------------------------------------------------------------- |
| 전체 톤     | `bg-gray-50`, `c-card`, `rounded-xl`, `shadow-sm` 중심의 기본 Tailwind 화면 |
| 메인 Hero   | 파란 그라데이션 박스 안에 텍스트만 있음                                     |
| 검색 UI     | Hero와 분리된 단순 카드형 6개 입력 필드                                     |
| 차량 카드   | 카드형으로 전환되었지만 이미지, 가격, 배지, 버튼 정도의 기본 구성           |
| 상세 화면   | 2열 구조는 있으나 이미지 갤러리, 딜러 카드, 시각적 깊이는 제한적            |
| 인증 화면   | 중앙 단일 카드형 로그인/회원가입                                            |
| 관리자 화면 | 사이드바 없이 요약 카드, 탭, 테이블만 있음                                  |
| 공통 스타일 | `c-btn`, `c-input`, `c-card` 등 기능적 공통 클래스 중심                     |

## 5. 차이가 생긴 이유

| 원인                                             | 설명                                                                                                                                                          | 다음 작업에서의 보정                                                                                                |
| ------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------- |
| 성공 기준이 기능 전환에 가까웠음                 | 7단계 문서의 핵심 완료 기준은 daisyUI 제거, 순수 Tailwind 전환, 카드 목록, 상담 UI 뼈대였다. 디자인 이미지와의 픽셀 수준 또는 분위기 수준 매칭 기준은 약했다. | 계획서에 화면별 시각 기준, 색상, 여백, 카드 밀도, 이미지 사용 기준을 명시한다.                                      |
| 목표 디자인 이미지가 시스템 토큰으로 풀리지 않음 | 파란색, 그림자, 라운드, 입력 높이, 이미지 비율, 관리자 사이드바 같은 기준이 코드 스타일 토큰으로 분해되지 않았다.                                             | `style.css`에 버튼, 패널, 입력, 배지뿐 아니라 `surface`, `hero`, `market-card`, `admin-shell` 계열 패턴을 정의한다. |
| 차량 이미지가 Hero에 적극 사용되지 않음          | 목표 이미지는 첫 화면에서 차량 사진이 브랜드 인상을 만든다. 현재는 파란 박스 Hero라 자동차 서비스 느낌이 약하다.                                              | Hero에 실제 차량 이미지 또는 기본 차량 이미지를 크게 배치하고, 검색 패널을 Hero 내부/하단에 겹치는 형태로 설계한다. |
| 관리자 화면 구조가 미완성에 가까움               | 목표 이미지는 사이드바 기반 관리자 화면인데 현재는 단일 페이지 탭 UI다.                                                                                       | 관리자 화면은 짙은 사이드바 + 요약 카드 + 최근 상담 테이블 구조로 재설계한다.                                       |
| 인증 화면의 브랜드 패널이 없음                   | 목표 이미지는 로그인/회원가입 화면에 차량 이미지와 브랜드 소개 패널이 있어 서비스성이 강하다.                                                                 | 로그인/회원가입을 2열 브랜드 패널 구조로 개선하고 모바일에서는 단일 카드로 접는다.                                  |
| 카드와 버튼이 너무 일반적임                      | 현재 `c-card`, `c-btn-primary`는 깔끔하지만 자동차 마켓 특유의 고급감은 부족하다.                                                                             | 카드 이미지 비율, hover, 스펙 아이콘, 가격 타이포, CTA 위치를 구체화한다.                                           |
| 기존 기능 유지 부담이 컸음                       | 7단계는 UI 외에도 상담방 API와 신규 컴포넌트까지 포함해 범위가 넓었다.                                                                                        | 이번 작업은 API, DB, 인증, Socket.io 로직을 변경하지 않고 프론트 UI에 집중한다.                                     |

## 8. 구현 범위

이번 11단계에서는 기능 로직을 바꾸지 않고 UI 표현을 개선한다.

| 화면           | 작업 방향                                                   |
| -------------- | ----------------------------------------------------------- |
| 공통 스타일    | `style.css`의 디자인 토큰과 공통 클래스 확장                |
| Header         | 목표 이미지처럼 얇고 밝은 상단 내비게이션, 모바일 메뉴 개선 |
| 첫 화면        | 차량 이미지 Hero + 검색 패널 + 추천 매물 카드 섹션          |
| 차량 카드      | 이미지 중심 카드, 가격 강조, 스펙 아이콘, 상담 CTA 추가     |
| 차량 상세      | 이미지 갤러리형 레이아웃, 스펙 리스트, 딜러 카드, CTA 개선  |
| 로그인         | 좌측 폼 + 우측 차량 브랜드 패널, 모바일 단일 카드           |

| 차량 등록/수정 | 단계형 또는 좌측 섹션 내비게이션이 있는 업무형 폼           |
| 상담방 목록    | 검색 입력, 차량 썸네일, 상대방, 마지막 메시지, 상태 뱃지    |

## 9. 변경하지 않는 항목

| 항목                              | 이유                                         |
| --------------------------------- | -------------------------------------------- |
| Express API 경로                  | `/api/cars`, `/api/users`, `/api/chats` 유지 |
| MongoDB 컬렉션 구조               | UI 작업이므로 데이터 구조 변경 불필요        |
| Firebase Authentication 설정      | 인증 로직은 이미 구현됨                      |
| Socket.io 이벤트 이름             | 요구사항의 이벤트 이름 유지                  |
| Render 단일 Web Service 배포 구조 | 현재 배포 기준 유지                          |
| 외부 이미지 스토리지              | 1차 구현은 `/uploads` 유지                   |
| 새 UI 패키지                      | 순수 Tailwind CSS 기준 유지                  |

## 10. 작업 순서

1. 색상 콘셉트를 사용자에게 확인받는다.
2. 필요하면 placeholder/hero 차량 이미지 사용 방식을 확인받는다.
3. `style.css`에 디자인 토큰과 공통 UI 패턴을 확장한다.
4. Header와 전체 레이아웃 배경을 먼저 정리한다.
5. 첫 화면 Hero, 검색 패널, 차량 카드 목록을 목표 이미지 기준으로 재작성한다.
6. 차량 상세 화면을 이미지 갤러리와 딜러 CTA 중심으로 개선한다.
7. 로그인/회원가입 화면을 브랜드 패널 포함 구조로 개선한다.
8. 차량 등록/수정 폼을 딜러 업무 화면처럼 정리한다.
9. 상담방 목록과 채팅 화면을 실시간 상담 서비스답게 정리한다.
10. 관리자 화면을 사이드바 기반 대시보드 스타일로 개선한다.
11. 375px, 768px, 1280px 기준 반응형을 확인한다.
12. `rg`로 daisyUI 계열 class가 다시 들어오지 않았는지 확인한다.
13. `npm.cmd --prefix frontend run build`와 `npm.cmd run build`로 검증한다.
14. 완료 후 `docs/steps/2026-06-06-11-modern-ui-redesign.md`, `docs/pr/2026-06-06-11-modern-ui-redesign-pr.md`, `docs/progress.md`를 갱신한다.

## 14. 이번 계획의 결론

7단계 작업은 daisyUI 제거와 기능 화면 전환은 달성했지만, 목표 디자인 이미지의 핵심인 차량 이미지 중심 Hero, 고급 카드 밀도, 인증 브랜드 패널, 관리자 사이드바, 색상 시스템까지는 구현 기준으로 충분히 내려오지 않았다.

11단계에서는 UI를 다시 “Modern Car Marketplace”로 끌어올리기 위해 색상과 화면별 시각 기준을 먼저 확정하고, API와 데이터 구조는 유지한 채 프론트엔드 표현 계층을 집중 개선한다.
